# Feature: pane-detectors (компакция / воркфлоу / ошибки)

**Цель.** Хуков на эти события нет — хаб вытаскивает их скрейпом пейна в `pollScreens`:
компакция (прогресс + «готово»), запущенный workflow (имя + N/M агентов), и баннеры ошибок/авторизации,
которые иначе юзер не увидит.

**Код.** `parseCompaction` / `parseWorkflow` / `parseError` (`tmux-ops.ts`, чистые),
`handleCompaction` / `handleWorkflow` / `handleErrors` (`hub.ts`), `compactMessages`/`workflowMessages`/`lastError`.

**Предусловие.** Контейнер, живая сессия.

---

## Юниты (есть, настоящие) ✅
`tests/core.test.ts`:
- `parseCompaction` — реальные форматы (с elapsed и без), идл-пейн → undefined, и **анти-ложняк**:
  слова «Compacting conversation» как СОДЕРЖИМОЕ скроллбэка без соседней шкалы не считаются.
- `parseError` — сигнатуры (`API Error`, `Invalid API key`, `Please run /login`, billing, `*_error`),
  срез ⏺/●-буллета, только хвост экрана (проза про ошибку не триггерит).
- `parseWorkflow` — имя воркфлоу + `N/M agents done` из статус-строки.
- `parseContextPct`, `paneIsWorking` — там же.

## Позитив

**PD1. 🎯 Компакция: прогресс → «готово».** `[СПЕЦ]`
- Шаги: `/compact` в живой сессии.
- Ожидаемо: в пейне «✻ Compacting conversation…», хаб постит статус компакции и по завершении
  «✅ Компакция готова».
- Смотреть: пейн, чат.

**PD2. Ошибка/авторизация в пейне → сообщение в топик.**
- Шаги: в пейне появляется баннер (`API Error…`, `Please run /login`).
- Ожидаемо: хаб один раз выносит его в чат (дедуп по `lastError`).
- Смотреть: чат, пейн.

**PD3. Workflow: имя + N/M агентов.**
- Шаги: агент запускает Workflow.
- Ожидаемо: самообновляемое сообщение с именем воркфлоу и счётчиком агентов.
- Смотреть: чат.

## Негатив
**PD4. Анти-ложняк:** разговор ПРО компакцию/ошибку (слова в скроллбэке без живой шкалы/в прозе)
не должен порождать статус-сообщений. Покрыто юнитами (`parseCompaction`/`parseError` — только хвост
экрана + требование соседней шкалы).

# Feature: voice (STT in / TTS out)

**Цель.** Входящие голосовые авто-транскрибируются (текст заменяет сообщение); `reply(voice:true)`
шлёт ответ голосом (TTS). Всё через OpenAI audio-эндпоинты, без ffmpeg (Telegram `.oga` = Ogg/Opus,
байт-совместим с TTS-выводом и `sendVoice`). Без `OPENAI_API_KEY` — voice молча выключен.

**Код.** STT/TTS в `hub.ts`/`stub.ts`, `STT_*`/`TTS_*` env, `doReply(voice)`.

**Предусловие.** `OPENAI_API_KEY` в state `.env` (в контейнере добавлен временно для теста).

---

## Позитив

**V1. Входящее голосовое → STT → агент отвечает.** `[СПЕЦ]`
- Шаги: прислать voice-note в топик.
- Ожидаемо: хаб транскрибирует, текст идёт агенту как сообщение; агент отвечает по смыслу.
- Fail-режим: голос проигнорен / не транскрибирован.
- Смотреть: пейн (агент получил текст), чат.

**V2. `reply(voice: true)` → голосовой ответ (TTS).** `[СПЕЦ]`
- Шаги: попросить агента ответить голосом.
- Ожидаемо: в чат приходит voice-note (TTS), не текст (или + текст).
- Смотреть: API (тип сообщения = voice/audio), чат.

## Негатив

**V3. Без ключа — voice выключен (текст всё равно работает).**
- Ожидаемо: `reply(voice:true)` без ключа → уходит текстом; входящий голос → не транскрибируется
  (или падает на fallback). Не крэш.
- Смотреть: чат.

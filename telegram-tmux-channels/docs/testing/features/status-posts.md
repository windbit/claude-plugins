# Feature: status-posts (PerTurnEditablePost)

**Цель.** Один самообновляемый пост на биндинг за ход для: подагентов (🤖 Агенты), тудушек
(📋 Задачи, `TaskCreate`/`TaskUpdate`), вызовов `Skill`. Готовые элементы остаются с ✅ (пост =
история хода, не только снапшот). Один пост шлётся ОДИН раз (msgId=-1 резервирование), дальше
редактируется на месте; новый ход = новое сообщение внизу.

**Код.** `PerTurnEditablePost` (send-once→edit, reservation guard, per-tracker turnEnded),
`subagent-hook.ts` → hub; `subPost`/`taskPost`/`todoPost`/`skillPost`.

**Предусловие.** Контейнер, живая сессия, хуки плагина засижены (в контейнере — entrypoint).

---

## Позитив

**SP1. 🎯 TodoWrite → один самообновляемый пост 📋, без дублей.**
- Шаги: агент делает `TaskCreate` (3 задачи) и `TaskUpdate` (одну → done).
- Ожидаемо: ОДНО сообщение 📋 Задачи, редактируется на месте; выполненная — с ✅; НЕ несколько
  сообщений (reservation guard — баг, который Stage 1 чинил: task/skill не имели резервирования).
- Fail-режим: N сообщений на N обновлений (double-send race).
- Смотреть: API (кол-во сообщений с 📋 = 1, текст обновляется), `screenlog.jsonl`.

**SP2. Подагенты → 🤖 Агенты пост.**
- Шаги: агент запускает Task-субагента.
- Ожидаемо: один 🤖 пост, статус агента дот→✅ по завершении.
- Смотреть: API.

**SP3. Skill-вызов → skill-пост.**
- Шаги: агент вызывает Skill.
- Смотреть: API.

## Негатив / швы

**SP4. Новый ход → новое сообщение (не дописывает старое).**
- Шаги: два отдельных хода с тудушками.
- Ожидаемо: у каждого хода свой 📋 пост (turnEnded → fresh).
- Смотреть: API.

**SP5. Несколько трекеров в одном ходу не воруют друг у друга.**
- Шаги: ход с субагентом И тудушками И скиллом.
- Ожидаемо: 3 независимых поста, каждый живёт своей жизнью (был баг «субагент невидим под скиллом»).
- Смотреть: API.

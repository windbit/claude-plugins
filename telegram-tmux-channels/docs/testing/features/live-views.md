# Feature: live-views (`/screen`, `/last`)

**Цель.** Живой просмотр пейна сессии в Telegram: `/screen` — самообновляемое фото (headless-chrome
рендер ANSI), `/last` — то же текстом (дайджест пейна). Обе: одна на пейн, кнопка «Закрыть» (удаляет),
авто-стоп рефреша через 3 мин, такт 5с.

**Предусловия.** Контейнер `clod-tmux` поднят; топик 3 (`sandbox`) забинден; живая подключённая
сессия claude в пейне (`/resume`, дождаться `claude` в `tmux list-panes`). Драйв — телеграм-аккаунтом
в тестовую группу (reply в msg 3 = топик 3).

**Код.** `startLiveScreen` / `refreshLiveScreen` / `stopRefreshing` / `closeLiveScreensForPane`
(`hub.ts`), `paneDigest` / `parseOpsCommand` (`tmux-ops.ts`), `renderScreenPng`.

---

## Позитив

**P1. `/last` → живой текст-дайджест.**
- Шаги: `/last` в топик с живой сессией.
- Ожидаемо: приходит `📄 <pane> · время` + `<pre>дайджест</pre>` + кнопка «✖️ Закрыть»; сама `/last` подчищается; таймстемп сдвигается каждые ~5с.
- Fail-режим: пусто/не приходит; не обновляется; мусор из рамок в дайджесте.
- Смотреть: Telegram API (`list_messages`/`list_inline_buttons`), Telegram-Web (рендер `<pre>`), hub-лог.

**P2. `/screen` → живое фото (нужен chrome).**
- Шаги: `/screen` при наличии chrome.
- Ожидаемо: фото пейна + «Закрыть», caption с таймстемпом тикает; при смене пейна фото перерисовывается.
- Fail-режим: не перерисовывается при изменении; caption не тикает.
- Смотреть: Telegram-Web (само фото!), API.

**P3. Кнопка «Закрыть» удаляет.**
- Шаги: нажать «✖️ Закрыть».
- Ожидаемо: сообщение удалено, `answerCallbackQuery`=«Закрыто».
- Смотреть: API (`list_messages` — сообщения нет).

**P4. Авто-стоп через 3 мин.**
- Шаги: оставить `/last` на >3 мин.
- Ожидаемо: рефреш встал, приписка «обновление остановлено», кнопка «Закрыть» ещё работает.
- Смотреть: API (таймстемп перестал расти + note), hub-лог.

## Негатив / швы

**N1. `[СПЕЦ]` нет chrome → `/screen` падает в текст.**
- Шаги: chrome отсутствует (контейнер), `/screen`.
- Ожидаемо: тот же live-view, что `/last` (текст), не ошибка.
- Fail-режим: молчание / краш.
- Смотреть: API, hub-лог. ✅ подтверждено на прогоне (контейнер без chrome).

**N2. `[ЭКСКЛЮЗИВ]` несколько `/screen`/`/last` на один пейн → one-per-pane.**
- Шаги: `/last`, затем ещё `/last` (и `/screen`) в тот же топик.
- Ожидаемо: прежнее живое окно этого пейна удаляется, остаётся ровно одно.
- Fail-режим: N параллельных циклов; **битая картинка** (корень: temp-файл `screen-<pane>-<ms>` при совпадении мс). → `regression-checklist` 2026-07-18.
- Смотреть: API (старое сообщение исчезло), Telegram-Web (нет дублей). ✅ подтверждено (msg48 удалён при втором `/last`).

**N3. `/last` без живой сессии.**
- Шаги: `/last` когда claude не подключён.
- Ожидаемо: `⚠️ Нет живой сессии. Попробуй /resume.`
- Смотреть: API.

**N4. `[СПЕЦ]` рестарт хаба при открытом live-view.**
- Шаги: открыть `/last`, рестартнуть хаб.
- Ожидаемо (текущее): live-view НЕ персистится (таймер в памяти) — сообщение застывает, кнопка «Закрыть» после рестарта мертва (нет в `liveScreens`).
- Fail-режим/вопрос UX: мёртвая кнопка молчит. Кандидат на улучшение — либо персистить (как пикеры), либо на старте до-закрывать осиротевшие. **Открытый вопрос, не баг.**
- Смотреть: hub-state.json (live-views там нет), API.

# Feature: session-lifecycle

**Цель.** Поднятие/переподнятие/остановка сессий: `/resume` (--continue), `/new` (свежая),
`/restart` (graceful /exit → relaunch, продолжает разговор), `/stop` (graceful, без relaunch).
Хаб создаёт tmux (имя по папке), прокликивает стартовые промпты. Не стартует вторую сессию, если
claude уже жив в папке. На буте `reviveBoundSessions` поднимает забинденные сессии.

**Код.** `spawnSession`/`restartSession`/`stopSession`, `buildLaunch`, `ensureTmuxSession`,
`ackStartupPrompts`, `reviveBoundSessions`, гвард «уже запущено».

**Предусловие.** Контейнер, биндинг топик 3.

---

## Позитив (частью ✅ по ходу сессии)

**SL1. `/resume` / `/new` / `/stop`.** ✅ — многократно за сессию (spawn, стоп, revive).
**SL2. Авто-ack стартовых промптов.** ✅ (= PB5): свежие сессии проходят trust/dev без пикера.
**SL3. Revive на буте.** ✅ (набл.): рестарт хаба → `reviveBoundSessions` поднимал сессию.

**SL4. `/restart` — graceful перезапуск, продолжает разговор.**
- Шаги: живая сессия с историей → `/restart`.
- Ожидаемо: сессия корректно завершилась и переподнялась (--continue/--resume), история цела,
  агент отвечает в контексте прошлого.
- Смотреть: чат (сообщения restart), пейн (claude перезапустился), проверка контекста вопросом.

## Негатив / швы

**SL5. Не-двойной-старт.** `[СПЕЦ]`
- Шаги: сессия жива → `/resume` (или `/new`).
- Ожидаемо: хаб не форкает вторую сессию (гвард «claude уже запущен в папке»), одна сессия.
- Fail-режим: вторая сессия/форк разговора.
- Смотреть: пейн (кол-во claude-процессов), чат.

**SL6. `/restart` мёртвой сессии.**
- Шаги: сессия вниз → `/restart`.
- Ожидаемо: поднимает заново (как /resume), без ошибки.
- Смотреть: чат, пейн.

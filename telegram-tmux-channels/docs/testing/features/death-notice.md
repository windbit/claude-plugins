# Feature: death-notice (💀)

**Цель.** Если tmux/процесс claude пропал БЕЗ явного `/restart`/`/stop`, хаб после грейса шлёт
💀-уведомление в топик (а не молча замолкает). Штатные разрывы (`/restart` помечает
`expectedDisconnect`) — без 💀.

**Код.** `notifyUnexpectedDeath`, `expectedDisconnect`, `DEATH_GRACE_MS` (5с), socket `close`.

**Предусловие.** Контейнер, живая сессия.

---

## Позитив

**DN1. 🎯 Неожиданная смерть → 💀 после грейса.** `[СПЕЦ]`
- Шаги: убить процесс claude (SIGKILL), НЕ через `/stop`/`/restart`.
- Ожидаемо: сокет стаба закрылся → не в `expectedDisconnect` → через ~5с без реконнекта → в топик
  «💀 Сессия оборвалась неожиданно … Напиши что-нибудь — переподнимется, или /resume».
- Fail-режим: тишина (молча замолчал) — баг, который фича и чинит.
- Смотреть: чат (💀), hub-лог.

## Негатив / швы

**DN2. Штатный `/stop`/`/restart` — без 💀.**
- Шаги: `/stop` (или `/restart`).
- Ожидаемо: НЕТ 💀 (разрыв ожидаемый, `expectedDisconnect`).
- Смотреть: чат (нет 💀; для /stop — «🛑 Сессия остановлена»).
- Наблюдение: за сессию много раз делал `/stop`/`/restart` — 💀 не всплывал. ✅ по наблюдению.

**DN3. Реконнект в грейсе → без 💀.**
- Шаги: краткий разрыв стаба с быстрым реконнектом (<5с).
- Ожидаемо: 💀 не шлётся (грейс дождался реконнекта).
- Смотреть: чат.

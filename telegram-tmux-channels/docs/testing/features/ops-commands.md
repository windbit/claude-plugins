# Feature: ops-commands

**Цель.** Управление живой сессией из чата: `/compact`, `/clear`, `/esc` (прерывание + дренаж
очереди ввода), `/enter` (сабмит того, что в строке), `/status`, `/model`. Не форвардятся в сессию —
перехватываются `handleOps`.

**Код.** `parseOpsCommand`, `handleOps` (диспатч), `sendKeys`.

**Предусловие.** Контейнер, живая сессия.

---

## Позитив (частью ✅ по ходу)

**OC1. `/status`** ✅ — многократно (папка/tmux/claude/лимиты/контекст).
**OC2. `/model`** ✅ (= PB2) — нативный пикер кнопками.

**OC3. `/esc` — прерывание живого хода + дренаж очереди.** `[СПЕЦ]` 🎯 (часть прод-фикса)
- Шаги: дать агенту долгую задачу → `/esc`.
- Ожидаемо: «⎋ Esc отправлен (+ очередь ввода очищена)»; ход прерван (Escape×3 250ms + Ctrl-U),
  агент не досчитывает; строка ввода чистая.
- Fail-режим: не прерывает; следующая из очереди сразу стартует (прод-раннэвей).
- Смотреть: пейн (прерван), чат.

**OC4. `/compact`** — компакция контекста.
- Шаги: `/compact`.
- Ожидаемо: «🗜 /compact отправлен», в пейне пошла компакция.
- Смотреть: пейн, чат.

**OC5. `/enter`** — сабмит строки ввода.
- Шаги: (в строке что-то есть, напр. недосабмиченный /compact) → `/enter`.
- Ожидаемо: голый Enter, сабмитит содержимое строки.
- Смотреть: пейн.

## Негатив / швы

**OC6. `/clear`** — очистка истории (деструктив).
- Шаги: `/clear`.
- Ожидаемо: «🧹 История очищена», сессия с чистым контекстом.
- Смотреть: пейн, проверка контекст-вопросом (не помнит прошлое).

**OC7. ops на мёртвой сессии.**
- Шаги: сессия вниз → `/compact`/`/esc`/`/model`.
- Ожидаемо: «⚠️ Нет живой сессии. Попробуй /resume.» (не авто-спавн для этих).
- Смотреть: чат.

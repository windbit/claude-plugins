# Feature: context-badge

**Цель.** Когда заполнение контекст-окна сессии достигает порога, к ответу агента добавляется
строка «⚠️ Контекст: NN%» — чтобы юзер видел, что скоро компакция. Порог — `TELEGRAM_CONTEXT_WARN_PCT`
(дефолт 80), `0` выключает. Процент берётся скрейпом статус-строки пейна (`parseContextPct`).

**Код.** `CONTEXT_WARN_PCT`, врезка в `doReply` (`hub.ts` ~303), `parseContextPct` (`tmux-ops.ts`).

⚠️ **Доккдрейф:** README/CLAUDE.md говорят «prepend … к ответу», а код **дописывает снизу**
(`text = \`${text}\n\n⚠️ Контекст: ${pct}%\``) — намеренно, чтобы не отодвигать сам ответ. Поправить доки.

---

## Юнит (есть, настоящий) ✅
`parseContextPct` в `tests/core.test.ts` — процент после пирога `○◔◑◕●` из статус-строки, только хвост экрана.

## Позитив

**CB1. 🎯 Порог достигнут → бейдж在 ответе.** `[СПЕЦ]`
- Шаги: поднять хаб с `TELEGRAM_CONTEXT_WARN_PCT=1` (низкий порог, чтобы гарантированно сработало) →
  задать агенту вопрос → он отвечает через `reply`.
- Ожидаемо: в конце ответа строка «⚠️ Контекст: NN%», NN совпадает с процентом в пейне.
- Смотреть: чат, статус-строка пейна.

## Негатив

**CB2. `0` выключает бейдж.** `[СПЕЦ]`
- Шаги: хаб с `TELEGRAM_CONTEXT_WARN_PCT=0` → ответ агента.
- Ожидаемо: бейджа нет вообще.
- Смотреть: чат.

**CB3. Ниже порога — бейджа нет.**
- Ожидаемо: при дефолтных 80 и контексте ~10% ответы чистые (наблюдалось всю сессию, но отдельного
  прогона не делал → ⬜).

## Лог прогона
- **2026-07-20 (проход 1)** — **фича проверена НЕ полностью, статус 🟡.**
  - Поднял хаб с `TELEGRAM_CONTEXT_WARN_PCT=1` (проверил, что переменная реально в окружении хаба).
  - **CB-guard ✅ (настоящий негатив):** при пороге 1 (сработал бы на любом pct ≥1) бейдж
    **не появился** — потому что в футере пейна этой сессии **вообще нет индикатора контекста**
    (только `⏵⏵ bypass permissions on … ← for agents`), `parseContextPct` → undefined. То есть
    «нет данных → нет бейджа», ложного бейджа не рисуется. Это подтверждено прогоном.
  - **CB1 ⬜ (не удалось воспроизвести):** чтобы бейдж сработал, в футере нужен числовой процент
    (вида `○ 12%`). В контейнерной сессии он не рендерится даже после накачки контекста; в других
    пейнах (напр. прод-agentek) он виден — значит зависит от сессии/наполнения. Нужен прогон на
    сессии с видимым индикатором.
  - **CB2 ⬜:** «0 выключает» в текущих условиях неотличимо от «нет pct» — отдельного вывода не даёт.
- **2026-07-20 (проход 2, по подсказке Ромы) — CB1 ✅ ДОКАЗАН на проде, без правки конфига:**
  - Контейнер оказался НЕ той средой: под его токеном строка плана (`○ NN%  █░░ M%  ⏱ …`) вообще
    не рендерится, поэтому pct недоступен в принципе. На прод-пейнах она есть: `◔ 26%`, `○ 11%`,
    `○ 7%`, `◕ 87%`.
  - **Парсер на живом проде:** `parseContextPct(capture-pane %6)` → **26** ✅.
  - **Сам бейдж на живом проде:** в `screenlog.jsonl` (tg_out, финальный payload) реально уходили
    «⚠️ Контекст: 93% / 85% / 84% / 82% / 81%» — все ≥ дефолтного порога 80. Фича работает в бою.
- **Вывод:** CB1 ✅ (прод, ground truth), guard ✅ (контейнер), парсер ✅. Остаётся ⬜ только CB2
  («0 выключает»). Статус → 🟢.
- **Урок метода:** отрицательный результат в харнессе ≠ фича сломана — сначала проверь, та ли среда.
  Здесь харнесс физически не мог показать pct, а прод показывал всё это время.
